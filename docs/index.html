What is Brick?

An OOP language with GDScript-inspired syntax that compiles directly to C. Designed for games, real-time systems, and any domain where performance and memory control matter.

Compiles to C

Your .brc code becomes readable C — then gcc turns it into a native binary. No VM, no interpreter, no overhead. Full C interop with include, link, and extern fn.

Block Memory

No garbage collector, no stack for user data. Everything lives in named blocks you control. Declare block game = 64MB, allocate with @game, reset everything with game.reset().

Hot Reload

Swap code without stopping your program. Powered by dlopen + inotify. Edit, save, see results instantly. Double-buffered — zero visible stutter.

Compile-Time Macros

Generate code with macro, compute at compile time with build, stamp output with emit. Zero runtime cost — macros expand before type-checking.

Windows Native Port

Full Windows support via MinGW-w64. VirtualAlloc/CRITICAL_SECTION, LoadLibrary/ReadDirectoryChangesW, and a Win32 window library. All 159 codegen tests pass on Windows. CI pipeline included.

Fixed-Width Types

i8/i16/i32/i64, u8/u16/u32/u64, f32/f64, usize/isize. Compile-time overflow checking, literal suffixes (42u8, 3.14f64), and strict type promotion rules.

C Interop

Call any C library directly: include "math.h", extern fn sqrt(f64) -> f64. Generate bindings from headers with brick bind.

Code Example

Interfaces, blocks, macros, and fixed-width types — all working together.

Brick (.brc) 
package DEMO

using IO

block global = 256MB
block temp   = 8KB

macro assert(condition, msg) {
    if !($condition) {
        print("ASSERT: {0}", $msg)
    }
}

interface Drawable {
    fn draw()
}

struct Circle {
    u32 id
    String name
    f32 radius

    fn Circle(u32 i, String n, f32 r) {
        id = i
        name = n
        radius = r
    }

    fn draw() {
        print("Circle #{0} radius={1}", id, radius)
    }
}

fn main() {
    assert(1 + 1 == 2, "math is broken")
    Circle c = Circle(1u32, "Small", 5.0f32) @global
    c.draw()
}

Macros: Code That Writes Code

Macros expand before type-checking — zero runtime cost. Use macro for templates, build for compile-time computation, and emit to generate output.

Swap Macro

macro swap(a, b) {
    __tmp = $a
    $a = $b
    $b = __tmp
}

swap(x, y)  // expands inline

Parameters are untyped — they hold expressions. $ inserts the argument. __ vars get unique names (gensym).

Build + Emit

build {
    dmg = [10, 25, 50, 100]
    for i = 0; i < 4; i++ {
        emit {
            if weapon == i {
                hp -= dmg[i]
            }
        }
    }
}

All for loops unroll at compile time. Only the emit blocks become real code.

Type Reflection

build {
    v = Vec2
    name   = v.name     // "Vec2"
    size   = v.size     // 8
    fields = v.fields   // ["x","y"]
}

Inspect type names, sizes, and field lists at compile time. Generate serializers and debug printers automatically.

Hygiene

// Macro __i becomes __i__1
// No collision with user code
macro twice(body) {
    __i = 0
    while __i < 2 { $body; __i++ }
}

__i = 100     // different variable
twice(print("hi"))
print(__i)  // still 100

Automatic name uniquing. Max 64 recursion levels. $ outside macro/build is a compile error.

Features

Everything you need for high-performance systems programming — nothing you don't.

OOP with Structs

Structs with methods, constructors, inheritance (extends), and interfaces. No this, no name shadowing. Clean GDScript-inspired syntax.

Compile-Time Macros

macro templates, build comptime evaluation, emit code generation. Swap variables, generate enums, serialize structs — all at compile time.

Block Memory

Declare named memory blocks (8KB – 256MB+). All allocations go into blocks. No malloc/free per object — just block.reset() clears everything.

Hot Reload

Edit code while it runs. dlopen + inotify detect changes, reload .so files, remap functions atomically. Zero downtime iteration.

Fixed-Width Types

i8..i64, u8..u64, f32/f64, usize/isize. Compile-time overflow checking, widening rules, literal suffixes. Predictable sizes on every platform.

C Interop

include "sdl2.h", extern fn, *T pointers. Call any C library. brick bind generates .brc bindings from headers.

GDB Integration

#line directives map every line back to .brc. Pretty-printers show blocks. Custom commands: info blocks, block <name>.

TUI Visualizer

Ncurses dashboard showing block memory usage and allocations in real time. brick --visualize file.brc or --attach <pid>.

VS Code Extension

Syntax highlighting, LSP (completions, hover, diagnostics), and memory-view panel that plots block allocations during debug.

Packages

Hierarchical package NAME.SUB, using import, private/public visibility. Each package can be a separate .so for hot reload.

Pool Allocator

Types ≤ 64 bytes automatically use a pool allocator (O(1) free). Larger types use the bump allocator (~3 ns). Both live inside blocks.

TLS Blocks

Thread-local current block via __thread (GCC) / __declspec(thread) (MinGW). Zero-contention allocations across threads — no locks, no atomics, no cache-line bouncing.

Windows Native

VirtualAlloc blocks, CRITICAL_SECTION thread safety, LoadLibrary hot reload, ReadDirectoryChangesW file watching, CreateWindow Win32 window library, brc-run.bat helper. Full CI pipeline.

Optimizations

7 techniques applied — from compile-time math to thread-local blocks.

🔧 Constant Folding

Pre-computes constant expressions (60 * 1000 → 60000) so the CPU never has to.

⚡ Inline Hints

Every non-main function gets always_inline. Zero call overhead for methods and small functions.

📏 SIMD Alignment

Float/f64 fields get aligned(16/32) so GCC emits aligned SSE/AVX moves — up to 2× faster math.

📦 Pool Allocator

Types ≤ 64 bytes use O(1) pool_alloc. Larger types use the bump allocator. Both inside blocks.

🧵 TLS Blocks

Thread-local current block via __thread. Each thread allocates without locks — zero contention.

🔄 Double-Buffer

Hot reload with zero pause. Block double-buffering swaps atomically — no frame skip, no stutter.

📊 PGO Builds

Profile-guided optimization: instrument, run benchmarks, rebuild. Up to 15% faster code.

Quick Start

From zero to running in four steps.

1

Clone

Get the repository and enter the project directory.

git clone https://github.com/nonunknown777/brick.git
cd brick
2

Build

Brick uses SCons. Build the compiler and runtime in one command.

scons                    # Linux
scons target=windows     # Windows (MinGW-w64)
3

Write

Create a .brc file with structs, methods, macros, and blocks.

touch game.brc
4

Compile & Run

Use the brick CLI — no manual gcc needed.

build/brick run game.brc
